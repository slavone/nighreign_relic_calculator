# Elden Ring Nightreign — Relic Probability Calculator

## Overview

A single self-contained HTML + JS file that calculates the probability of rolling a relic with a specific set of desired effects. No backend, no dependencies. Can be pasted into CodePen, hosted on GitHub Pages, or shared as a raw HTML file.

---

## Data Model

Relic effect data is sourced from the game's `AttachEffectParam` and `AttachEffectTableParam` data files (via the community GitHub repo `ip1259/Elden-Ring-Nightreign-Legal-Relic-Generator`). It is compiled into a JSON array embedded in the HTML file:

```json
[
  {
    "compatibilityId": 7000000,
    "groupName": "Vigor",
    "effects": [
      { "id": 7000000, "name": "Vigor +1" },
      { "id": 7000001, "name": "Vigor +2" },
      { "id": 7000002, "name": "Vigor +3" }
    ]
  },
  ...
]
```

### Key facts from data extraction

- **77 groups**, **356 total rollable effects** in the common relic pool.
- Groups are defined by `compatibilityId` in the game data. Only one effect per group can appear on a relic.
- Notable large groups: attack power (71 effects, compatId 100), character-specific (58, compatId 900), spell schools (14, compatId 800), HP-on-attack by weapon type (24), FP-on-attack (24), weapon discovery (33).
- Effects like "Attacks Inflict [Status]" are **not** in the random pool — they only appear on unique named relics.
- Rollable effects list is in `data/relic_groups.json` (generated by `parse_relics.mjs`).

### Relic quality

Relic quality determines the number of effect slots (not a random roll):

| Quality | Effects |
|---|---|
| Delicate | 1 |
| Polished | 2 |
| Grand | 3 |

The Grand quality shop is the primary farming target; the calculator defaults to Grand.

## Probability Model

### Assumptions

- K groups are selected uniformly at random (without replacement) from all G groups, where K is the relic quality (1/2/3).
- Within each selected group, one effect is chosen uniformly at random.
- Relic color (blue / yellow / green / red) is an independent uniform 1/4 roll.
- Note: the game data contains non-uniform weights per effect, but these are not yet publicly documented in a usable form. Uniform probability is used as the stated assumption.

### Formula

Let:
- `G` = 77 (total number of groups in the rollable pool)
- `K` = relic quality (1 = Delicate, 2 = Polished, 3 = Grand)
- `n` = number of desired effects specified by the user (1 ≤ n ≤ K)
- `s_i` = number of effects in the group containing the i-th desired effect

```
P(all n desired effects present) =
  C(G−n, K−n) / C(G, K) × ∏(1/s_i)
```

- `C(G−n, K−n) / C(G, K)` — probability that all n desired groups are among the K groups chosen for the relic.
- `∏(1/s_i)` — probability the correct specific effect is picked within each desired group.

### Special case

If the user selects two desired effects from the **same group** (same `compatibilityId`), the probability is 0 — impossible combination.

### Color factor

- Color specified → multiply P by `1/4`
- Color = "Any" → no adjustment (color doesn't affect effects)

## UI Structure

Single HTML page, plain HTML + CSS + JS, no framework.

### Inputs

| Input | Type | Options |
|---|---|---|
| Relic Quality | Dropdown | Grand (default) / Polished / Delicate |
| Color | Dropdown | Any / Blue / Yellow / Green / Red |
| Desired effect 1 | Two dropdowns | Group → Effect within group |
| Desired effect 2 | Two dropdowns (revealed by "Add effect") | Group → Effect |
| Desired effect 3 | Two dropdowns (revealed by "Add effect") | Group → Effect |

- "Add effect" button reveals the next slot, up to the max allowed by quality (e.g., max 2 slots for Polished).
- Once a group is selected in one slot, it is removed from the other slots' group dropdowns (enforces one-effect-per-group constraint).
- A "Remove" button on each slot (except the first) lets the user clear it.

### Output

- Live-updating (no submit button): probability recalculates on every input change.
- Displayed as: `Probability: 2.34%`
- If inputs are incomplete (no effect selected), output is hidden.
- If an impossible combination is selected (two effects from the same group), show: `Impossible combination (same group)`.

### Style

Minimal CSS — readable on desktop, no external stylesheets or fonts.

## Out of Scope

- Mobile layout optimization
- Saving/sharing specific builds
- Simulation / Monte Carlo verification (formula is sufficient)
- Any backend or database
- Per-effect non-uniform probability weights (unknown distribution)
